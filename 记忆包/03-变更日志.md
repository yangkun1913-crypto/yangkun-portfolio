## 2026-06-05 — 图片加载彻底修复 + 头像更新

### 图片加载机制重写（第4次修复）
- **问题**：109张图片同时发起HTTP请求，浏览器同域名并发连接数限制（~6个），大量请求排队超时导致部分图片不显示
- **尝试过的方案**：
  1. InterpObserver + `loading="lazy"` → 冲突，快速滚动时丢失回调
  2. `requestAnimationFrame` 分批 → 不可靠，tab后台时停止
  3. 直接 `img.src` 全量加载 → 连接池拥堵，12张超时
- **最终方案**：`setTimeout` 同步+分批加载，每批6张（匹配浏览器连接池）、间隔200ms
  ```js
  // 分批加载核心：同一帧内连续加载6张 → setTimeout 200ms → 下一批6张
  function loadAt(i) {
    img.src = img.dataset.src;  // 立即触发加载
    if ((i + 1) % 6 === 0) setTimeout(() => loadAt(i + 1), 200);
    else loadAt(i + 1);
  }
  ```
- 移除所有 `loading="lazy"` 和 IntersectionObserver 代码
- **验证结果**：109/109 全部加载，零失败

- 添加 `.nojekyll` 文件，禁用 Jekyll 处理（加速部署）

## 已解决的问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 部分图片不显示（12张） | 109张图同时发起HTTP请求，浏览器并发连接数限制（~6/域名） | `setTimeout` 分批加载，每批6张间隔200ms |
| GitHub Pages 部署旧版缓存 | CDN缓存 + 缺少 `.nojekyll` | 添加 `.nojekyll` 文件，禁用Jekyll处理 |
| 所有图片不显示（JS报错） | `renderGallery()` 在 `observer` 定义前调用，触发 TDZ 错误 | 把 `observer` 定义移到 `renderGallery()` 之前 |
| Chrome `file://` 不加载图片 | Chrome 安全策略 | 使用 HTTP 服务器（`python3 -m http.server`） |
| 图片变形 | 声明的宽高比与实际像素不符 | 用 `sips` 扫描实际像素比后更新 |
| 文件名含空格加载失败 | URL 中空格未正确处理 | 文件名空格改为下划线 |
| `encodeURI()` 导致 file:// 协议出错 | 双重编码 | 移除所有 `encodeURI()` 调用 |
