# 部署指南

## 本地预览

双击 `预览网站.command`，或手动执行：
```bash
cd "/Users/xx/Desktop/个人网站"
python3 -m http.server 8080
# 浏览器打开 http://localhost:8080/index.html
```

## 方案A：GitHub Pages（推荐，免费）

### 前置条件
- GitHub 账号（github.com 免费注册）
- 本机已安装 git（系统自带）

### 部署步骤

1. **在 GitHub 网站创建新仓库**
   - 仓库名建议：`photography-portfolio` 或 `portfolio`
   - 设为 Public（公开）
   - **不要**勾选 "Add a README file"

2. **推送代码**
   ```bash
   cd "/Users/xx/Desktop/个人网站"
   git remote add origin https://github.com/你的用户名/仓库名.git
   git branch -M main
   git push -u origin main
   ```

3. **开启 GitHub Pages**
   - 打开仓库的 Settings → Pages
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "main"，文件夹选 "/ (root)"
   - 点击 Save
   - 等待 1-2 分钟，页面会显示网址：`https://你的用户名.github.io/仓库名/`

4. **后续更新**
   ```bash
   cd "/Users/xx/Desktop/个人网站"
   git add -A
   git commit -m "更新内容描述"
   git push
   ```

## 方案B：Netlify（免费，最简单）

1. 注册 netlify.com 账号
2. 在 Dashboard 点击 "Add new site" → "Deploy manually"
3. 将整个 `个人网站/` 文件夹拖入上传区域
4. 自动获取 `xxx.netlify.app` 域名

## 方案C：飞书妙搭（免费，需飞书账号）

```bash
# 安装 lark-cli（首次）
npm install -g @larksuite/cli

# 配置认证
lark-cli config init --new
lark-cli auth login --domain apps

# 创建并发布
lark-cli apps +create --name "许洋堃摄影作品集" --app-type HTML
lark-cli apps +html-publish --app-id <返回的app_id> --path ./deploy
lark-cli apps +access-scope-set --app-id <app_id> --scope public --require-login=false
```

## 部署注意事项

- `.gitignore` 已排除原始图文件夹 `作品集/`（仅上传压缩后的 `作品集-web/`）
- 总部署大小约 25MB，GitHub Pages 限制 1GB，绰绰有余
- `index.html` 中图片路径使用**相对路径**（`作品集-web/...`），不要改为绝对路径
- GitHub Pages 启用后改代码只需 `git push`，自动更新
