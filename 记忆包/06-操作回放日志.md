## R06 · 2026-06-05 · 图片加载修复（最终方案 ✓）

### 触发原因
R04 的直接全量加载导致连接池拥堵，12 张图片排队超时。

### 执行指令
```javascript
// 核心方案：setTimeout 分批加载
// 每批 6 张（= 浏览器连接池大小），批间间隔 200ms

function loadAt(i) {
  if (i >= items.length) return;
  const img = items[i].querySelector('img');
  if (img && img.dataset.src) {
    img.src = img.dataset.src;
    img.removeAttribute('data-src');
    items[i].classList.add('visible');
  }
  if ((i + 1) % 6 === 0) {
    setTimeout(() => loadAt(i + 1), 200);
  } else {
    loadAt(i + 1);
  }
}
loadAt(0);
```

### 具体操作
1. 恢复 `img.dataset.src` 机制
2. 移除直接 `img.src = photo.url`
3. 实现 `loadAt()` 递归分批加载
4. BATCH=6（匹配浏览器 HTTP 连接池），DELAY=200ms
5. 同一批内同步连续执行（不设延迟），批间 setTimeout 200ms

### 修改文件
- `index.html`（最终分批加载方案）

### 验证结果
- 本地 Playwright：109/109 ✓
- 线上 GitHub Pages：109/109 ✓（零失败）
- 所有分类筛选正常

### 最终架构
```
DOM创建 → 所有img带datase-src → loadAt(0)
  ↓
第1批(1-6)：同步设置src → 200ms → 第2批(7-12)...共19批
  ↓
图片onload → 移除shimmer → .loaded class
```
